Voici une explication complète de l'agent ag.py (analyse de l'action ATW à la Bourse de Casablanca) :                                          
                        
  ---                                                                                                                                            
  🎯 Objectif global                                                                                                                                                                                                                                                                              
  Cet agent analyse l'action ATW (Attijariwafa Bank) en combinant 7 sources de données différentes pour produire :                               
  - Un verdict (ACHAT / CONSERVER / VENDRE)
  - Une prédiction de trading (4 semaines)
  - Une prédiction d'investissement (12 mois)

  ---
  📋 Étapes du pipeline (pas à pas)

  Étape 1 — Chargement des données depuis PostgreSQL

  L'agent lit 7 tables de la base :

  ┌─────────────────┬──────────────────────────────┬──────────────────────────────────────────┐
  │     Source      │            Table             │                 Contenu                  │
  ├─────────────────┼──────────────────────────────┼──────────────────────────────────────────┤
  │ Marché          │ bourse_daily                 │ Prix, volume, capitalisation (252 jours) │
  ├─────────────────┼──────────────────────────────┼──────────────────────────────────────────┤
  │ Carnet d'ordres │ bourse_orderbook             │ Bid/Ask sur 5 niveaux                    │
  ├─────────────────┼──────────────────────────────┼──────────────────────────────────────────┤
  │ Indicateurs     │ technicals_snapshot          │ RSI, MACD, Bollinger                     │
  ├─────────────────┼──────────────────────────────┼──────────────────────────────────────────┤
  │ Macro           │ macro_morocco                │ PIB, inflation, dette                    │
  ├─────────────────┼──────────────────────────────┼──────────────────────────────────────────┤
  │ Fondamentaux    │ fondamental_snapshot         │ PER, ROE, dividende                      │
  ├─────────────────┼──────────────────────────────┼──────────────────────────────────────────┤
  │ Valorisations   │ models_result.json (fichier) │ DCF, DDM, Graham                         │
  ├─────────────────┼──────────────────────────────┼──────────────────────────────────────────┤
  │ News            │ news                         │ Articles récents avec score              │
  └─────────────────┴──────────────────────────────┴──────────────────────────────────────────┘

  Étape 2 — Validation qualité des données (DataQualityChecker)

  Vérifie :
  - Valeurs manquantes (CRITICAL)
  - Sauts de prix > 20 % (suspect)
  - Données obsolètes (> 3 jours)
  - Bid ≥ Ask (carnet inversé = erreur)

  Étape 3 — Calcul des indicateurs marché

  ATR (Average True Range) — mesure de volatilité :
  TR = max(High−Low, |High−Close_prev|, |Low−Close_prev|)
  ATR(14) = moyenne(TR sur 14 derniers jours)

  Volatilité 20 jours :
  σ = écart-type(rendements_quotidiens) × 100

  Performance vs moyenne mobile :
  Prix_vs_MA20 = (Prix_actuel / MA20 − 1) × 100

  Étape 4 — Features du carnet d'ordres

  Order Imbalance (OI) — pression acheteur/vendeur :
  OI = (Volume_bid − Volume_ask) / (Volume_bid + Volume_ask)
  - OI > 0 → pression acheteuse
  - OI < 0 → pression vendeuse

  Spread bid-ask en bps :
  Spread = (Ask₁ − Bid₁) / Bid₁ × 10 000

  Z-score OI (déviation par rapport à la session) :
  Z = (OI_actuel − moyenne_OI) / écart_type_OI
  - |Z| > 2 → signal fort

  Étape 5 — Détection du régime de marché (detect_market_regime)

  Logique :
  - Calcule rendement moyen sur 1 sem / 1 mois / 3 mois
  - Compare prix vs MA20 et MA50
  - Décide :
    - BULL si rendement > +1 % et au-dessus MA20 et faible volatilité
    - BEAR si rendement < −1 % et sous MA20 et forte volatilité
    - SIDEWAYS sinon

  Étape 6 — Prédiction Trading (4 semaines)

  Cibles ATR :
  Zone_entrée = [Prix − 0.5×ATR ; Prix + 0.5×ATR]
  Objectif    = Prix + 1.5 × ATR
  Stop loss   = Prix − 2.0 × ATR

  Simulation Monte Carlo (1000 trajectoires sur 20 jours) :
  S(t+1) = S(t) × exp(drift + σ × Z)
  où Z ~ N(0,1)
     drift = μ − 0.5 × σ²

  À partir des 1000 prix finaux :
  - Probabilité de rendement positif = % trajectoires > prix actuel
  - VaR 95 % = −percentile(rendements, 5)
  - Intervalle de confiance 90 % = [percentile(5), percentile(95)]

  Étape 7 — Prédiction Investissement (12 mois)

  Juste valeur = moyenne des 4 modèles (DCF, DDM, Graham, Relatif)

  Potentiel (upside) :
  Upside = (Juste_valeur − Prix_actuel) / Prix_actuel × 100

  Seuils dynamiques (s'adaptent au régime + volatilité) :
  Seuil_achat  = 15 % × (vol/2.5) × mult_régime × facteur_incertitude
  Seuil_vente  = −10 % × ...

  Multiplicateurs régime :
  - BULL : achat 0.8, vente 1.2 (plus facile d'acheter)
  - BEAR : achat 1.3, vente 0.7 (plus difficile)

  Étape 8 — Bloc Evidence + LLM

  L'agent compile tout dans un texte avec identifiants (ex: [MKT-PRICE], [VAL-DCF], [NEWS-1]) et l'envoie au LLM Groq (gpt-oss-120b) qui doit    
  produire un JSON structuré avec des citations valides.

  Étape 9 — Validation des citations

  Vérifie que chaque [ID] cité par le LLM existe vraiment dans le bloc. Sinon, fallback déterministe.

  ---
  ✅ Avantages

  1. Multi-sources — combine 7 dimensions (très complet)
  2. Probabiliste — Monte Carlo donne une vraie distribution, pas juste un point
  3. Adaptatif — seuils qui s'ajustent au régime de marché
  4. Traçable — chaque conclusion est citée avec un ID
  5. Robuste — fallback déterministe si le LLM échoue
  6. Validation qualité — détecte automatiquement les données suspectes

  ---
  ❌ Inconvénients

  1. Dépendance LLM — Groq peut être lent, indisponible ou halluciner
  2. Monte Carlo simpliste — suppose des rendements log-normaux i.i.d. (irréaliste lors de crises)
  3. Régime à 3 états seulement — trop simple (manque crash, recovery, etc.)
  4. Moyenne simple des 4 modèles de valorisation — devrait être pondérée par leur précision historique
  5. Pas de backtest intégré — on ne sait pas si les seuils donnent vraiment de bons résultats
  6. Drift Monte Carlo basé sur 1 sem + 1 mois seulement — biais récent fort
     (✅ horizons désormais pondérés correctement, mais l'historique reste court)

  ---
  ⚠️ Limites importantes

  ┌──────────────────────────────────┬──────────────────────────────────────────────────────────────────┐
  │              Limite              │                              Impact                              │
  ├──────────────────────────────────┼──────────────────────────────────────────────────────────────────┤
  │ np.random.seed(42) fixé          │ Mêmes résultats à chaque exécution → faux sentiment de précision │
  ├──────────────────────────────────┼──────────────────────────────────────────────────────────────────┤
  │ Volatilité historique uniquement │ Ne capte pas les chocs futurs (volatilité implicite ignorée)     │
  ├──────────────────────────────────┼──────────────────────────────────────────────────────────────────┤
  │ Pas de corrélations              │ Macro et marché traités indépendamment                           │
  ├──────────────────────────────────┼──────────────────────────────────────────────────────────────────┤
  │ Single stock only                │ ATW seulement, pas de comparaison sectorielle                    │
  ├──────────────────────────────────┼──────────────────────────────────────────────────────────────────┤
  │ Données macro Maroc partielles   │ unemployment_rate / interest_rate non scrapés (source absente)   │
  ├──────────────────────────────────┼──────────────────────────────────────────────────────────────────┤
  │ LLM température 0.2              │ Encore une part d'aléa dans la thèse narrative                   │
  ├──────────────────────────────────┼──────────────────────────────────────────────────────────────────┤
  │ Bug IMF connu (✅ filtré)        │ Valeurs aberrantes désormais coupées au LOAD (None au lieu de…) │
  └──────────────────────────────────┴──────────────────────────────────────────────────────────────────┘

  ▎ demande au LLM de rédiger la narration en citant les preuves — combinant rigueur quantitative et explication en langage naturel.
